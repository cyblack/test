# user2.cpp Lagrange 방향성 분석

## 배경: 왜 방향이 중요한가?

전환비용 `|pv - v| × 0.01` 때문에, v를 올릴 때와 내릴 때 **함수 자체가 달라진다**.

```
올릴 때(v ≥ pv):  비용 = +(v - pv)·0.01   → 올릴수록 비용 증가
내릴 때(v < pv):  비용 = -(v - pv)·0.01   → 내릴수록 비용 증가
```

따라서 `v = pv` 지점을 경계로 **두 개의 서로 다른 포물선**이 존재한다.

---

## 두 포물선의 수식

### Case A 포물선 (v ≥ pv 영역)

```
f_A(v) = S·v - c·P·v²/10000 - c·0.01·v
       = (S - c·0.01)·v - c·P·v²/10000
```

- 전환비용이 `+(v-pv)·0.01` → v에 대한 1차항이 **감소**
- 꼭짓점: `v_up = (S - c·0.01) × 5000 / (c·P)`

### Case B 포물선 (v < pv 영역)

```
f_B(v) = S·v - c·P·v²/10000 + c·0.01·v
       = (S + c·0.01)·v - c·P·v²/10000
```

- 전환비용이 `-(v-pv)·0.01` → v에 대한 1차항이 **증가**
- 꼭짓점: `v_dn = (S + c·0.01) × 5000 / (c·P)`

### 핵심 차이

```
v_dn의 1차 계수: S + c·0.01  (더 큼)
v_up의 1차 계수: S - c·0.01  (더 작음)

→ v_dn > v_up   (항상!)
→ Case B 포물선이 Case A보다 항상 오른쪽에 꼭짓점을 가짐
```

---

## 그래프: 3가지 상황

### 상황 1: v_up ≥ pv → 올리는 게 이득

```
  L(v)
   ▲
   │          ○ v_up (Case A 꼭짓점)
   │         ╱ ╲
   │        ╱   ╲
   │       ╱     ╲          ← Case A 포물선 (v ≥ pv 에서 유효)
   │      ╱       ╲
   │     ╱         ╲
   │    ╱
   │  ╱
   │╱
───┼──────┬──────────┬──────── v
   0     pv        v_up

   [  Case B 영역  |  Case A 영역  ]
                    ← 여기서 v_up 발견!
```

**v_up이 pv 오른쪽** → Case A 영역 안에 꼭짓점이 존재

→ pv에서 출발해서 **오른쪽으로 올라가는 구간**이 있음

→ v_up까지 올리면 최대 → **v = v_up 채택**

---

### 상황 2: v_dn < pv → 내리는 게 이득

```
  L(v)
   ▲
   │    ○ v_dn (Case B 꼭짓점)
   │   ╱ ╲
   │  ╱   ╲
   │ ╱     ╲                ← Case B 포물선 (v < pv 에서 유효)
   │╱       ╲
   │         ╲
   │          ╲
   │           ╲
   │            ╲
───┼──────┬──────┬────────── v
   0    v_dn    pv

   [  Case B 영역  |  Case A 영역  ]
       ← 여기서 v_dn 발견!
```

**v_dn이 pv 왼쪽** → Case B 영역 안에 꼭짓점이 존재

→ pv에서 출발해서 **왼쪽으로 올라가는 구간**이 있음

→ v_dn까지 내리면 최대 → **v = v_dn 채택**

---

### 상황 3: v_up < pv, v_dn ≥ pv → 안 움직이는 게 이득

```
  L(v)
   ▲
   │              ★ pv (현재 위치 = 최적!)
   │             ╱ ╲
   │            ╱   ╲
   │     ╱╲   ╱     ╲   ╱╲
   │    ╱  ╲ ╱       ╲ ╱  ╲
   │   ╱   v_up     v_dn   ╲
   │  ╱  (A의 꼭짓점) (B의 꼭짓점) ╲
   │ ╱                           ╲
───┼──────────────┬──────────────── v
   0             pv

   [   Case B 영역   |   Case A 영역   ]
   [B의 꼭짓점이 여기 →] [← A의 꼭짓점이 여기]
   [    오른쪽에...    ] [    왼쪽에...       ]
         범위 밖!              범위 밖!
```

**v_up이 pv 왼쪽**: Case A 영역(v ≥ pv)에서는 이미 내리막
- pv에서 오른쪽으로 가면? Case A 포물선은 이미 꼭짓점을 지남 → **내려감**

**v_dn이 pv 오른쪽**: Case B 영역(v < pv)에서는 아직 오르막
- pv에서 왼쪽으로 가면? Case B 포물선은 아직 꼭짓점 전 → **내려감**

```
    ← 왼쪽(내리기): 손해     pv     오른쪽(올리기): 손해 →
                             ★
                           최적!
```

→ **어느 쪽으로 움직여도 손해** → v = pv 유지

## 실제 합성 그래프 (두 포물선을 이어붙인 모습)

실제로 최적화하는 함수는 두 포물선의 **유효 구간만 이어붙인** 형태:

```
  L(v)
   ▲
   │
   │         ╱╲          ╱╲
   │        ╱  ╲        ╱  ╲
   │       ╱  B ╲      ╱ A  ╲
   │      ╱      ╲    ╱      ╲
   │     ╱        ╲  ╱        ╲
   │    ╱          ╲╱          ╲
   │   ╱           pv           ╲
   │  ╱         (꺾이는 점)       ╲
───┼──────────────┬──────────────── v
   0             pv

   [  Case B 유효  |  Case A 유효  ]
```

- `v = pv`에서 **꺾임** (미분 불연속)
- 왼쪽은 Case B 포물선, 오른쪽은 Case A 포물선
- 전체 함수는 여전히 **오목(concave)** → 최대점이 유일

---

## 꺾임의 물리적 의미

```
pv = 50 인 상태에서:

v = 49로 내리면:  만족도 S 감소  +  전환비용 0.01 발생
v = 50 유지하면:  만족도 변화 없음  +  전환비용 0
v = 51로 올리면:  만족도 S 증가  +  전환비용 0.01 발생
```

- v=50 양옆에 **0.01의 장벽**이 있음
- 이 장벽을 넘을 만큼 S 변화가 크면 → 움직임 (상황 1, 2)
- 장벽보다 S 변화가 작으면 → 제자리 (상황 3)

---

## 코드 대응 정리

```cpp
double c   = 0.01 + mu;                  // 전체 벌금 계수
double tr  = c * 0.01;                    // 전환비용 벌금 (장벽 크기)
double den = c * fPow[i] / 5000.0;        // 전력 벌금 (포물선 곡률)

double vU = (S[i] - tr) / den;            // Case A 꼭짓점 (올릴 때)
double vD = (S[i] + tr) / den;            // Case B 꼭짓점 (내릴 때)

if      (vU >= pv[i]) v = vU;             // 상황1: 올리는 게 이득
else if (vD <  pv[i]) v = vD;             // 상황2: 내리는 게 이득
else                  v = pv[i];           // 상황3: 장벽 안에 있음, 유지
```

| 변수 | 의미 |
|------|------|
| `S[i]` | v를 1 올렸을 때 만족도 이득 |
| `tr` | 전환비용 장벽의 크기 (μ에 비례) |
| `den` | 포물선의 곡률 (P가 클수록 가파름) |
| `vU` | "올리기 포물선"의 정상 |
| `vD` | "내리기 포물선"의 정상 |
| `pv[i]` | 두 포물선의 경계 (꺾이는 점) |
